023/**
024 * Using this annotations at type-level of a junit 5 test enable the myfaces-test-based
025 * test-framework for unit-tests. The configuration is implemented using some kind of decorator
026 * pattern (roughly). The actual configuration relies on two sources:
027 * <ul>
028 * <li>It scans for the annotations of type {@link JsfTestConfiguration} instantiates the
029 * corresponding classes and configures the environment accordingly</li>
030 * <li>It checks whether the actual test-class implements one of {@link ApplicationConfigurator},
031 * {@link BeanConfigurator}, {@link RequestConfigurator} or {@link ComponentConfigurator} and calls
032 * the corresponding methods accordingly, <em>after</em> the configurator derived by the
033 * annotations</li>
034 * </ul>
035 * <h3>Using</h3>
036 * <p>
037 * Simple Sample: Use a preconfigured JSF-context for the test providing
038 * BasicApplicationConfiguration
039 * </p>
040 *
041 * <pre>
042 * <code>
043     &#64;EnableJsfEnvironment
044     &#64;JsfTestConfiguration(BasicApplicationConfiguration.class)
045     class JSFEnableTest {
046</code>
047 * </pre>
048 *
049 * <p>
050 * Complex Sample: Use a preconfigured JSF-context with access to the context information like
051 * {@link FacesContext}, {@link ComponentConfigurator}, ... as defined within
052 * {@link JsfEnvironmentHolder} With this setup you can programmatically access the element
053 * </p>
054 *
055 * <pre>
056 * <code>
057     &#64;EnableJsfEnvironment
058     &#64;JsfTestConfiguration(BasicApplicationConfiguration.class)
059     class JsfSetupExtensionTest implements JsfEnvironmentConsumer {
060
061        &#64;Setter
062        &#64;Getter
063        private JsfEnvironmentHolder environmentHolder;</code>
064 * </pre>
065 *
066 * <p>
067 * In addition there is a new way of dealing with localized messages for unit-tests. In essence
068 * there is the {@link IdentityResourceBundle} configured: This is helpful for tests
069 * where you want to ensure that a certain message key is used to create a message but do not want
070 * to test the actual ResourceBundle mechanism itself. It will always return the given key itself.
071 * As default this mechanism is active, you can change this by setting
072 * {@link #useIdentityResourceBundle()}. If it is active it is used as well for resolving
073 * the MessageBundle. In case of many annotations of {@link EnableJsfEnvironment} are in the type
074 * hierarchy, the one on the most concrete Type will be chosen, usually on the actual unit-test.
075 * </p>
076 *
077 * @author Oliver Wolff
078 *
079 */
080@Documented
081@Retention(RUNTIME)
082@Target(TYPE)
083@ExtendWith(JsfSetupExtension.class)
084@Repeatable(EnableJsfEnvironments.class)
085public @interface EnableJsfEnvironment {
086
087    /**
088     * @return boolean
089     */
090    boolean useIdentityResourceBundle() default true;
091
092}

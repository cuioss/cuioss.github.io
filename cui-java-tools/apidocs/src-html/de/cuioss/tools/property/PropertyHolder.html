021/**
022 * <h2>Overview</h2>
023 * An instance of {@link PropertyHolder} provides runtime information for a specific BeanProperty.
024 * Under the hood it uses {@link Beans} tooling provided by the JDK and the utilities
025 * {@link PropertyUtil} an and {@link MoreReflection}. Compared to the standard tooling it is
026 * more flexible regarding fluent api style of bean / DTOs.
027 * <h3>Usage</h3>
028 * <p>
029 * The usual entry-point is {@link #from(Class, String)}. In case you want to create your own
030 * instance you can use the contained builder directly using {@link #builder()}
031 * </p>
032 * <p>
033 * Now you can access the metadata for that property, see {@link #getMemberInfo()},
034 * {@link #getName()}, {@link #getType()}, {@link #getReadMethod()}
035 * </p>
036 * <p>
037 * Reading and writing of properties should be done by {@link #readFrom(Object)}
038 * and {@link #writeTo(Object, Object)}. Directly using
039 * {@link #getReadMethod()} and {@link #getWriteMethod()} is more error-prone and less versatile.
040 * </p>
041 *
042 * <h2>Caution:</h2>
043 * <p>
044 * Use reflection only if there is no other way. Even if some of the problems are
045 * minimized by using this type. It should be used either in test-code, what we actually do, and
046 * not production code. An other reason could be framework code. as for that you should exactly know
047 * what you do.

147        requireNonNull(beanType);
148        requireNotEmptyTrimmed(attributeName);
149        try {
150            var info = Introspector.getBeanInfo(beanType);
151            var descriptor = mutableList(info.getPropertyDescriptors()).stream().filter(desc -> attributeName.equalsIgnoreCase(desc.getName())).findFirst();

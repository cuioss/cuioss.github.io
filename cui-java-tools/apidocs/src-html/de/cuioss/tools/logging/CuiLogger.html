012/**
013 * <p>
014 * Wrapper around java-util {@link Logger} that simplifies its usage. In
015 * addition it provides a similar api like slf4j. It is not meant to act as logging-facade like
016 * slf4j or jakarta-commons-logging. It only provides a little syntactic sugar for the built-in
017 * logger.
018 * </p>
019 * <h2>Obtaining a logger</h2>
020 * <p>
021 * {@code private static final CuiLogger log = new CuiLogger(SomeClass.class);}
022 * </p>
023 * <p>
024 * {@code private static final CuiLogger log = new CuiLogger("SomeLoggerName");}
025 * </p>
026 * <p>
027 * {@code private static final CuiLogger log = CuiLoggerFactory.getLogger();}
028 * </p>
029 * <h2>Logging</h2>
030 * <p>
031 * {@link CuiLogger} provides an implicit code guard, if used correctly. Used correctly hereby means
032 * to either use formatting with parameter or incorporating {@link Supplier} for generating the
033 * actual log-message. For other means of creating a message you still can use code guards.
034 * </p>
035 * <p>
036 * {@code log.trace("Parameter-type matches exactly '%s'", assignableSource);}
037 * </p>
038 * <p>
039 * {@code log.debug("Adding found method '%s' on class '%s'", name, clazz);}
040 * </p>
041 * <p>
042 * {@code log.info("Starting up application");}
043 * </p>
044 * <p>
045 * {@code // In order not to mess up with the ellipsis parameter}
046 * </p>
047 * <p>
048 * {@code // exceptions must be the first parameter}
049 * </p>
050 * <p>
051 * {@code log.warn(e, "Exception during lenientFormat for '%s'", objectToString); }
052 * </p>
053 * <p>
054 * {@code log.error(e, "Caught an exception"); }
055 * </p>
056 * <p>
057 * {@code log.info(() -> "Supplier can be used as well");}
058 * </p>
059 * <p>
060 * {@code log.error(e, () -> "Even with exceptions");}
061 * </p>
062 * <p>
063 * {@code log.trace(() -> "I will only be evaluated if the trace-level for is enabled");}
064 * </p>
065 * <h2>Formatting</h2>
066 * <p>
067 * Like slf4j there is a simple way of formatting log-messages. In addition to '{}' the
068 * formatting supports '%s' as well. At runtime it replaces the '{}' tokens with
069 * '%s' and passes the data to {@link MoreStrings#lenientFormat(String, Object...)} for
070 * creating the actual log-message. As a variant providing a {@link Supplier} works as well.
071 * </p>
072 *
073 * @author Oliver Wolff
074 *
075 */

082    /**
083     * @param clazz to be used for acquiring a concrete {@link Logger} instance.
084     *            Must not be null
085     */
086    public CuiLogger(Class<?> clazz) {
087        requireNonNull(clazz);
088        delegate = Logger.getLogger(clazz.getName());
089    }
090
091    /**
092     * @param name to be used for acquiring a concrete {@link Logger} instance.
093     *            Must not be null
094     */
095    public CuiLogger(String name) {
096        requireNonNull(nullToEmpty(name));
097        delegate = Logger.getLogger(name);
098    }
